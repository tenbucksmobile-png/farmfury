  BLOCK._ENSURETEXTURE(scene, type, w, h) — static
    · Standard 60×20 wood → 'wood'  (pre-built, no overhead)
    · Standard 60×30 stone → 'stone' (pre-built)
    · Any other size → generates 'type_WxH' texture on first use
    · Wood pattern: grain lines every 15px; Stone: mortar scaled to w/h
    · Mass and maxHP scaled proportional to area vs standard size

  LEVELLOADER CLASS (between Robot and Trebuchet in file order)
    loadLevel(idx)
      · _clearLevel() → spawn blocks/robots → birdsLeft=[...data.birds]
      · _buildQueueDisplay() → update HUD text, destroyCount=0
      · camera.setScroll(300,200) + trebuchet.draw() to reset view
    _clearLevel()
      · destroy activeAnimal, all blocks (img+crackGfx+hpGfx), all robots
      · _destroyQueueDisplay(); birdsLeft = []
    _destroyQueueDisplay()
      · destroys _baleGfx + _queueCircles only — does NOT touch birdsLeft
    _buildQueueDisplay()
      · Hay bale at screen (70,554): brown #7A5218, width=count×28+14
      · Per-bird Graphics circles at local (0,0), positioned at screen coords
        so tweening .x/.y animates them in screen space ✓
    consumeBird()
      · Shifts birdsLeft; tweens first circle (pos+alpha) → trebuchet tip
      · Screen-space target: (LAUNCH_X - cam.scrollX, LAUNCH_Y - cam.scrollY)
      · Slides remaining circles left: x = 70+14+i×28
      · Fades out hay bale after last bird (delay 420ms)
    get hasMoreBirds → birdsLeft.length > 0

  TREBUCHET CHANGES
    draw():  draws yellow circle r=10 at arm tip when hasMoreBirds && !activeAnimal
    fire():  guards !loader.hasMoreBirds; calls consumeBird(); calls draw()
             immediately after creating CluckAnimal to clear bird circle

  CLUCKANIMAL CHANGE
    destroy(): calls scene.trebuchet.draw() after setting activeAnimal=null
               so next-bird circle reappears on the sling

  SCENE CHANGES
    · levelNameText at screen (VW/2, 8), gold #FFD700, shows id + name + par
    · levelLoader created after trebuchet, before input handlers
    · keydown-N: loadLevel(currentIdx + 1), wraps with modulo
    · keydown-R: loadLevel(currentIdx)
    · No more manual spawnRobot() in create() — LevelLoader owns spawning
    · loadLevel(0) called at end of create()

  KEY TECHNICAL NOTE
    _destroyQueueDisplay does NOT reset birdsLeft. _clearLevel does.
    loadLevel sets birdsLeft then calls _buildQueueDisplay (which trusts it).
    This avoids double-setting and re-reading from LEVELS unnecessarily.
